Prompt

Create your own "Rozin Mirror" based on the work of Daniel Rozin, specifically his 'Software Mirrors' (although of course, the software mirror you create can be inspired by his physical installations). Basically, finish what we started in class on Tuesday, Feb 9.

If we were to reduce the Rozin Mirror project to a single prompt, it might be: Create your own "filter" that abstracts, distorts, or somehow manipulates a live webcam feed, and mirrors it back to the participant. This filter does not need to be "intelligent", it only has to translate/transform raw color information received from the webcam into some other aesthetic.

Guidelines

I feel that the expectations of this assignment were made clear in class as we worked on these. But just to be specific...

  1. Using p5.js, create a sketch that uses raw color information from a webcam feed to draw something to the p5.js canvas (for example, draw a square for every 20th pixel that changes size depending on how bright a pixel is). Feel free to draw on any experience you might have with p5.js.
  2. Move your code out of the p5.js web editor and into an actual web page! We covered this at the end of class on Tuesday (lecture 02, part 4), but I will be posting a standalone video showing this process, as this is something we'll be doing throughout the semester.

In class on Tuesday, we will be sharing our mirrors with each other. Since we aren't meeting in person, this show-and-tell will have us sharing our links with each other, and experiencing each others' work on our own computers.

How to submit (again, this isn't required to pass the class)

But also so that I have a link to your repository for future reference :) ) (as mentioned above, I'll be posting a video showing how to do this, but it will basically be a condensed version of the lecture 2, part 4 video).
